{\bfseries{The system was tested using texts of varying lengths and different shift values to evaluate the performance of each distance metric.}}

{\bfseries{Methodology}}
\begin{DoxyItemize}
\item Short texts (10–20 characters)
\item Long texts (200+ characters)
\item Multiple shifts (e.\+g., 5, 13, random values)
\item All three metrics applied to each case
\end{DoxyItemize}

{\bfseries{Observations}}
\begin{DoxyItemize}
\item Text length is critical:\+ Short texts often lead to incorrect decryption due to unreliable frequency distributions, while long texts significantly improve accuracy.
\item Chi-\/\+Squared Distance:\+ Most accurate overall, especially for longer texts. Best at detecting correct shifts.
\item Euclidean Distance:\+ Performs well but slightly less precise than Chi-\/\+Squared.
\item Cosine Distance:\+ Least reliable, sometimes identifies similar patterns but fails to detect the exact shift.
\end{DoxyItemize}

{\bfseries{Errors \& Edge Cases}} Decryption may fail in:\+
\begin{DoxyItemize}
\item Very short texts
\item Repetitive texts (e.\+g., "{}\+AAAAAA"{})
\item Texts with unusual letter distributions (missing common letters like \textquotesingle{}E\textquotesingle{} or \textquotesingle{}T\textquotesingle{}) These cases distort frequency analysis and reduce accuracy. 
\end{DoxyItemize}\hypertarget{index_autotoc_md6}{}\doxysection{\texorpdfstring{Conclusion}{Conclusion}}\label{index_autotoc_md6}
